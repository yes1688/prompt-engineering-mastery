# 文檔改寫任務分派指令

## 標準改寫任務模板

以下是給Claude Code agents的標準改寫任務提示：

### 基礎任務框架

```
你是專業的企業文檔改寫專家。請根據以下規範改寫指定文檔，讓非技術背景的企業員工能夠輕鬆理解和應用。

## 📋 改寫目標
將技術導向的提示工程文檔轉化為企業友善的培訓材料，適合跨部門員工學習使用。

## 📚 參考資料
1. 閱讀 `/mnt/datadrive/MyProjects/prompt-engineering-mastery/企業文檔標準規範.md` - 了解改寫標準
2. 參考 `/mnt/datadrive/MyProjects/prompt-engineering-mastery/01-提示工程基礎概念-企業版.md` - 了解改寫風格
3. 查看 `/mnt/datadrive/MyProjects/prompt-engineering-mastery/文檔分析報告.md` - 了解問題分析

## 🎯 具體改寫要求

### 1. 語言簡化
- 將專業術語轉換為日常商業語言
- 為每個必要的技術概念提供「💡 白話解釋」框
- 使用具體的商業場景舉例說明

### 2. 內容精簡
- 程式碼範例減少70%，僅保留3-5個最實用的核心範例
- 移除過於學術化的理論說明
- 聚焦於「如何使用」而非「如何運作」

### 3. 結構優化
- 採用企業文檔標準格式
- 增加實用的檢核清單和操作指南
- 提供立即可用的模板和範例

### 4. 受眾適配
- 面向一般企業員工（行銷、客服、人資、業務等）
- 強調商業價值和實際效益
- 提供跨部門協作的應用建議

## 📝 輸出要求
- 創建 `[原文檔名]-企業版.md` 檔案
- 保持原有的學習目標和核心概念
- 文檔長度控制在原版的60-70%
- 確保內容的專業性和準確性
```
