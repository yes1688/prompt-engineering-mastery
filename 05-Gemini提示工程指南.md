# Gemini 提示工程指南

> 基於Google官方文檔的Gemini最佳實踐指南

## 📋 概述

Gemini是Google開發的多模態大型語言模型，具有強大的文本、圖像和代碼理解能力。本指南基於Google官方文檔，提供針對Gemini模型的專業提示工程策略。

## 🎯 Gemini 模型特性

### 1. 多模態能力
- **文本理解**：卓越的自然語言處理
- **圖像分析**：可以分析和描述圖像內容
- **代碼生成**：支援多種程式語言
- **跨模態推理**：結合不同類型的輸入進行推理

### 2. 思維鏈優勢
Gemini在Chain-of-Thought提示上表現特別出色，特別是與**自一致性**（self-consistency）結合時。

### 3. 指令跟隨精度
Gemini能夠準確跟隨詳細的指令，對**具體、明確的提示**反應最好。

## 🚀 核心最佳實踐

### 1. 「Let's think step by step」- Gemini版本

#### 零樣本思維鏈
```
問題：一家商店原本有120個蘋果，上午賣出了40%，下午又進貨50個，晚上賣出了20個。現在還剩多少個蘋果？

Let's think step by step:

步驟1：計算上午賣出的蘋果數量
120 × 40% = 120 × 0.4 = 48個蘋果

步驟2：計算上午賣出後剩餘的蘋果
120 - 48 = 72個蘋果

步驟3：加上下午進貨的蘋果
72 + 50 = 122個蘋果

步驟4：減去晚上賣出的蘋果
122 - 20 = 102個蘋果

答案：現在還剩102個蘋果。
```

#### 少樣本思維鏈模式
```
以下是如何解決數學文字問題的範例：

範例1：
問題：Sarah有15顆糖果，她給了朋友1/3，然後又買了8顆。她現在有多少顆糖果？
思考過程：15的1/3是15÷3=5顆。給出5顆後剩下15-5=10顆。買了8顆後有10+8=18顆。
答案：18顆糖果。

範例2：
問題：一個班級有24名學生，其中2/3是女生。如果又有3名女生轉入，現在女生有多少名？
思考過程：原本女生人數是24×(2/3)=16名。轉入3名女生後，女生總數是16+3=19名。
答案：19名女生。

現在請解決這個問題：
問題：[你的數學問題]
```

## 🏗️ 提示結構設計

### 1. 清晰的上下文提供

#### 充分背景模式
```
背景脈絡：
我是一家中型電商公司的營運經理，公司年收入約2億元，主要銷售家居用品。
最近客戶投訴率上升了25%，主要集中在配送延遲和產品品質問題。

當前情況：
- 平均配送時間從3天增加到5天
- 退貨率從2%上升到4%
- 客戶滿意度從4.2分下降到3.6分（5分制）
- 競爭對手提供更快的配送服務

任務：
請分析這個問題的根本原因，並提供具體的改善策略。包含：
1. 問題分析（至少3個層面）
2. 解決方案（短期和長期）
3. 實施優先順序
4. 預期效果和風險評估

請用專業但易懂的語言回答，字數控制在800-1000字。
```

### 2. 具體詳細的指令

#### 詳細規格模式
```
寫作任務：撰寫產品評測文章

具體要求：
- 字數：1200-1500字
- 目標讀者：25-40歲的科技愛好者
- 語調：專業但親切，避免過於技術化的術語
- 結構：引言(10%) + 產品介紹(20%) + 詳細測試(50%) + 優缺點總結(15%) + 購買建議(5%)

內容要求：
- 包含至少5個具體的測試場景
- 提供與競品的比較
- 包含實際使用照片的描述
- 給出1-10分的評分及理由
- 提及價格性價比分析

寫作風格：
- 使用第一人稱經驗分享
- 包含具體數據和測量結果
- 避免過度推銷的語言
- 保持客觀和平衡的觀點

產品資訊：[提供產品詳細資訊]
```

### 3. 序列化任務處理

#### 複雜任務分解
```
大任務：設計完整的數位行銷策略

請按照以下順序完成每個子任務：

任務1：市場分析
- 分析目標市場規模和特徵
- 識別主要競爭對手
- 評估市場機會和威脅

任務2：目標受眾定義
- 基於任務1的結果，定義具體的目標受眾
- 建立用戶畫像
- 分析受眾的線上行為模式

任務3：渠道策略
- 基於前兩個任務的結果，選擇最適合的行銷渠道
- 為每個渠道制定具體策略
- 估算各渠道的投資回報率

任務4：內容策略
- 設計適合各渠道的內容類型
- 制定內容發布計劃
- 設定內容成效評估指標

任務5：整合與預算
- 整合所有策略成一個統一計劃
- 分配預算到各個渠道和活動
- 制定時程表和里程碑

請一次完成一個任務，每個任務完成後請確認是否可以進行下一個任務。
```

```
範例1：

範例2：
輸入：客戶抱怨「客服態度差，解決問題不積極」
輸出：{問題分類: "服務態度", 嚴重程度: 8, 處理部門: ["客服部", "人資部"], 回應策略: "道歉+內部調查+服務改善"}

範例3：
輸入：客戶抱怨「產品品質有問題，不如廣告描述」
輸出：{問題分類: "產品品質", 嚴重程度: 9, 處理部門: ["品管部", "行銷部"], 回應策略: "道歉+退換貨+品質保證"}

現在請處理：
輸入：客戶抱怨「網站下單困難，付款總是失敗」
```

### 2. 自一致性方法

#### 多路徑推理
```
請用3種不同的方法解決以下問題，然後比較結果：

問題：一個投資組合包含股票A（佔40%）、股票B（佔35%）、債券C（佔25%）。
如果股票A上漲15%，股票B下跌8%，債券C上漲3%，整體投資組合的收益率是多少？

方法1：分別計算每個投資項目的貢獻，然後相加
方法2：使用加權平均公式計算
方法3：用百分比變化的概念逐步計算

完成三種計算後，請比較結果並說明哪種方法最準確。
```

### 3. 迭代改進

#### 自我修正循環
```
首次回答：請回答這個商業問題：[問題描述]

自我檢查：請檢查上述回答是否：
1. 邏輯清晰且完整？
2. 考慮了所有重要因素？
3. 提供了可執行的建議？
4. 語言表達是否清楚？

修正版本：基於檢查結果，請提供改進後的回答。

最終確認：請確認這個版本是否已經足夠全面和準確。
```

### 2. 蘇格拉底式問答

#### 引導式探索
```
教學目標：讓學生理解[概念名稱]

蘇格拉底式教學規則：
1. 不直接提供答案
2. 通過問題引導學生思考
3. 根據學生回答調整後續問題
4. 鼓勵學生自己得出結論
5. 適時提供提示但不給出完整答案

學生當前理解水平：[描述學生背景]

請開始第一個引導性問題，並準備好根據不同可能的學生回答來設計後續問題。

範例問答流程：
問題1：「當你聽到[概念]這個詞時，你想到什麼？」
[根據回答調整] → 問題2：...
[繼續引導] → 問題3：...
```

### 2. 長度控制策略

#### 回應長度管理
```
簡短回應（100-200字）：
「請用2-3句話簡潔地回答：[問題]」

中等長度（300-500字）：
「請用一個段落詳細解釋：[問題]，包含主要觀點和一個具體例子。」

詳細回應（800-1200字）：
「請提供完整的分析，包含背景、分析、結論和建議，字數控制在800-1200字。」

超詳細回應（1500+ 字）：
「請撰寫一份完整的報告，涵蓋所有相關面向，提供深入分析和多個案例。」
```

## 🔍 常見問題解決

### 1. 回應不夠具體

#### 問題：回答過於籠統
```
❌ 模糊提示：
「請給我一些行銷建議」

✅ 具體提示：
「請為我們的有機食品線上商店提供3個具體的社交媒體行銷策略，
目標是在6個月內提升25-35歲女性客群的品牌認知度，
預算限制在每月5萬元以內，請包含具體的執行步驟和預期效果。」
```

### 2. 輸出格式不一致

#### 解決方案：明確格式要求
```
請嚴格按照以下JSON格式輸出：

{
  "分析結果": {
    "主要發現": ["發現1", "發現2", "發現3"],
    "風險評估": {
      "高風險": ["項目"],
      "中風險": ["項目"],
      "低風險": ["項目"]
    }
  },
  "建議行動": {
    "立即行動": ["行動1", "行動2"],
    "短期計劃": ["計劃1", "計劃2"],
    "長期目標": ["目標1", "目標2"]
  },
  "成功指標": ["指標1", "指標2", "指標3"]
}

請確保輸出完全符合此格式，不要添加額外說明。
```

### 3. 忽略重要限制

#### 解決方案：強調約束條件
```
重要限制條件（必須遵守）：
⚠️ 預算上限：絕對不能超過10萬元
⚠️ 時間限制：必須在3個月內完成
⚠️ 法規遵循：必須符合當地廣告法規
⚠️ 品牌調性：保持專業正式的風格

請在提供任何建議時，確保所有方案都在這些限制範圍內。
如果某個好想法超出限制，請說明如何調整以符合要求。
```
